第一章 Go 语言概述

1.1 Go 是什么

Go（又称 Golang）是 Google 开发的开源编程语言，2009年首次发布，2025年迎来16周年。

核心特性：

  • 静态类型 + 编译型：编译速度快，运行效率接近 C
  • 垃圾回收（GC）：Go 1.25 引入 Green Tea GC，Go 1.26 正式加入新 GC，延迟更低
  • 原生并发：goroutine + channel，轻松处理百万级并发
  • 简洁语法：25个关键字，学习曲线平缓
  • 跨平台编译：一条命令编译到 Linux/macOS/Windows/ARM 等十几个平台
  • 丰富的标准库：net/http、encoding/json、database/sql 等，开箱即用

1.2 Go 适合做什么

领域 典型应用
后端服务 API 服务器、微服务（Docker、Kubernetes 均为 Go 编写）
云原生 容器编排、Service Mesh、Serverless
网络工具 代理、网关、负载均衡
命令行工具 CLI 工具、构建工具
基础设施 Terraform、Prometheus、etcd、Consul

1.3 Go vs 其他语言

特性 Go Java Python Rust
编译速度 ⚡ 极快 慢 解释执行 慢
并发模型 goroutine 线程/协程 asyncio async/await
运行性能 优秀 良好 较慢 极佳
学习曲线 低 中 低 高
内存安全 GC GC GC 所有权系统
交叉编译 原生支持 需要配置 依赖环境 原生支持

第二十章 面试高频题

20.1 语言基础

Q: Go 中 make 和 new 的区别？

  • new(T) 返回 *T，分配零值内存
  • make 只用于 slice、map、channel，返回初始化后的引用类型

Q: slice 和 array 的区别？

  • array 固定长度，值类型
  • slice 动态长度，引用类型，底层指向 array

Q: channel 有缓冲和无缓冲的区别？

  • 无缓冲：发送和接收必须同时就绪（同步）
  • 有缓冲：发送可以在缓冲区未满时完成（异步）

Q: goroutine 和线程的区别？

  • goroutine 初始栈仅 2KB（线程通常 1-8MB）
  • goroutine 由 Go runtime 调度（M:N 模型）
  • 创建和切换成本极低

20.2 并发编程

Q: 什么情况下会 panic？

  • 数组越界、空指针解引用、向已关闭的 channel 发送、类型断言失败等

Q: context 的作用？

  • 超时控制、取消传播、请求级数据传递

Q: 如何避免 goroutine 泄露？

  • 确保 goroutine 有退出条件
  • 使用 context.WithCancel/WithTimeout
  • 使用 select 监听 context.Done()

20.3 内存与性能

Q: Go 的逃逸分析是什么？

  • 编译器决定变量分配在栈上还是堆上
  • 被外部引用的变量会逃逸到堆
  • go build -gcflags="-m" 查看逃逸分析

Q: Go 的 GC 是怎样的？

  • 并发标记清除（CMS）
  • Go 1.26 使用 Green Tea GC，延迟更低
  • 可通过 GOGC 和 GOMEMLIMIT 调优

20.4 实战设计

Q: 设计一个限流器？

  • 令牌桶（Token Bucket）
  • 漏桶（Leaky Bucket）
  • 滑动窗口（Sliding Window）
  • Go 标准库 golang.org/x/time/rate

Q: 如何实现优雅关闭？

  • 监听 os.Signal（SIGINT/SIGTERM）
  • 调用 http.Server.Shutdown(ctx)
  • 等待现有请求处理完成
