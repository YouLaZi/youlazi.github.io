智慧教学管理平台 技术文档

1. 项目概述

项目背景

传统教学管理面临课堂互动不足、实验环境搭建复杂、排课效率低等问题。本项目整合在线课堂、智能排课、在线IDE等功能，构建一站式智慧教学管理平台。

项目目标

  • 实现低延迟视频课堂（<200ms）
  • 提供在线编程实验环境
  • 开发智能排课算法
  • 构建三端协同的教学管理体系

项目成果

  • 🏆 省级大创项目
  • 🏆 中国计算机设计大赛国家级三等奖
  • 视频延迟 <200ms，支持 100+并发
  • QPS经压测优化从 200提升至800+
  • 三端完整交付：前端 + 后端 + 排课算法

3. 技术栈详解

技术 用途
Spring Boot 后端核心框架，RESTful API
Vue.js 前端框架，管理端/教师端/学生端
WebRTC 实时音视频通信，低延迟视频课堂
WebSocket IM即时通讯，实时消息推送
Docker 在线IDE沙箱环境，代码安全执行
MySQL 关系型数据存储
Redis 缓存、会话管理、消息队列
JMeter 性能压测工具

4. 核心功能模块

4.1 视频课堂模块（WebRTC）

  • 实时音视频：基于WebRTC的P2P通信
  • 屏幕共享：教师端共享屏幕进行演示
  • 白板功能：在线协作白板
  • 课堂录制：课程录像回放
  • 延迟优化：<200ms端到端延迟

4.2 在线IDE模块（Docker沙箱）

  • 在线编码：浏览器内代码编辑器
  • 沙箱执行：Docker容器隔离运行环境
  • 多语言支持：C/C++/Java/Python等
  • 实时反馈：运行结果即时返回

4.3 IM即时通讯模块

  • WebSocket长连接：实时消息收发
  • 群组聊天：课程群组、班级群组
  • 私聊：师生一对一沟通
  • 消息类型：文本、图片、文件

4.4 智能排课模块

  • 约束建模：教室、教师、课程、时间约束
  • 排课算法：遗传算法/回溯算法求解
  • 冲突检测：自动检测时间/教室/教师冲突
  • 手动调整：支持拖拽调课

4.5 教学管理模块

  • 课程管理、班级管理、学生管理
  • 作业发布与提交、自动批改
  • 成绩管理、考勤统计

5.2 JMeter压测与性能优化

压测结果：QPS从200提升至800+

优化措施：

优化项 方案
数据库 索引优化 + 慢SQL分析
缓存 Redis缓存热点数据
连接池 HikariCP参数调优
异步 WebSocket消息异步处理
并发 WebRTC连接池管理

5.3 Docker在线IDE

  • 基于Monaco Editor（VS Code内核）构建在线编辑器
  • Docker容器预装多种编程语言环境
  • 容器资源限制（CPU/内存/网络）
  • 容器生命周期管理（创建→使用→超时销毁）

6. 数据库设计

核心表结构

课程表 (course)

字段 类型 说明
id BIGINT 主键
name VARCHAR(200) 课程名称
teacher_id BIGINT 授课教师
schedule JSON 排课信息
classroom VARCHAR(50) 教室

消息表 (message)

字段 类型 说明
id BIGINT 主键
from_user BIGINT 发送者
to_user BIGINT 接收者
group_id BIGINT 群组ID(可选)
content TEXT 消息内容
type TINYINT 消息类型
created_at DATETIME 发送时间

7. 部署方案

  • 服务器：腾讯云轻量级应用服务器
  • 前端：Nginx部署Vue静态资源
  • 后端：SpringBoot JAR + Docker容器
  • 数据库：MySQL + Redis同机部署
  • WebRTC：STUN/TURN服务器中继
