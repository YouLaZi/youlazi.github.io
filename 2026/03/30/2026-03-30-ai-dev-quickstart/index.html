AI 应用开发快速入门

面向 Java 全栈开发者的 AI 应用开发指南，从概念到实战。

一、LLM 基础概念

什么是大语言模型（LLM）

大语言模型是基于 Transformer 架构、通过海量文本训练的神经网络，能理解和生成人类语言。核心能力：文本生成、问答、摘要、翻译、代码编写。

关键参数

参数 说明 建议值
Temperature 控制随机性，值越高输出越随机 代码生成 0，创意写作 0.7-1.0
Top-P 核采样，从概率累积前 P 的 token 中选 0.9
Max Tokens 单次生成的最大 token 数 按需设置
System Prompt 设定模型角色和行为规则 始终设置

Token 是什么

Token 是模型处理文本的最小单位。中文大约 1-2 个字 = 1 个 token，英文大约 4 个字符 = 1 个 token。计费按 token 数算，输入和输出都算。

二、RAG（检索增强生成）

为什么需要 RAG

LLM 的知识截止到训练数据，且会产生幻觉（编造内容）。RAG 让 LLM 先检索外部知识，再基于检索结果回答，解决知识过时和幻觉问题。

三、Agent（智能体）

核心概念

Agent = LLM + 记忆 + 工具。LLM 充当大脑，自主决定何时调用什么工具。

ReAct 模式

Function Calling

让 LLM 输出结构化的 JSON 来调用函数，而非自由文本。这是现代 Agent 的底层机制。

四、LangChain 快速入门

核心模块一览

模块 作用 类比
Model 封装 LLM API 调用 DAO 层
Prompt 管理提示词模板 SQL 拼接
Chain 串联多个步骤 Service 层
Agent LLM 自主决策调用工具 带脑子的 Service
Memory 对话历史管理 Session
Retriever 从向量库检索 搜索引擎

与 Java 类比：LangChain 就像 Spring，提供了一整套开发框架，各模块像 Bean 一样可以自由组合。

五、LlamaIndex 快速入门

与 LangChain 对比

维度 LangChain LlamaIndex

强项 Agent、多工具编排、复杂 Chain RAG、文档解析、知识图谱

适合场景 多工具 Agent 应用 文档问答、知识库

七、最佳实践

  1. Prompt 要明确：用 System Prompt 定义角色、输出格式、约束条件
  2. 分块参数：chunk_size 500-1000，overlap 10%-20%，按语义断句
  3. 成本控制：优先用 gpt-4o-mini / claude-haiku，复杂任务再用大模型
  4. 错误处理：API 调用务必加重试（tenacity 库）
  5. 评估闭环：用 RAGAS 等框架评估 RAG 效果，持续优化

八、常见问题

Q: LangChain 版本更新太快怎么办？
A: 锁定版本 pip install langchain==x.x.x，关注官方 Changelog。

Q: RAG 检索不准怎么办？
A: 优化分块策略 → 换更好的 Embedding 模型 → 用 Reranker 二次排序 → 调整检索数量 k。

Q: Agent 陷入死循环？
A: 设置最大迭代次数，添加超时机制，优化工具描述让 LLM 更容易选对工具。
