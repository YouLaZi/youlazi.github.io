CSOJ（智码）- AI辅助教学在线判题平台 技术文档

1. 项目概述

项目背景

随着高校编程教学规模的扩大，传统编程教学方式面临批改效率低、反馈不及时、缺乏个性化指导等问题。本项目旨在构建一个支持多语言、高可用的智能在线判题平台，为编程教学提供自动化的代码评测和AI辅助教学能力。

项目目标

  • 构建支持 9种编程语言 的在线判题系统
  • 实现 Docker沙箱隔离 的安全判题机制
  • 提供高可用的微服务架构，保障系统稳定运行
  • 通过CI/CD实现自动化部署，降低运维成本

项目成果

  • 优秀毕业设计 Top 5/2000+（全校2000+毕业生中排名前5）
  • 累计服务 500+学生，日均判题 1000+次
  • 编写 119个单元测试，保证代码质量
  • 三模块完整工程：前端、后端、代码沙箱独立部署

2. 系统架构

整体架构

系统采用 Spring Cloud微服务架构，分为三个核心模块：

架构设计亮点

  • 微服务拆分：用户、题目、判题、提交等服务独立部署，可独立扩展
  • 消息队列解耦：判题请求通过 RabbitMQ 异步处理，削峰填谷
  • 读写分离：MySQL主从复制，读请求分流到从库，降低主库压力
  • 沙箱隔离：每个判题任务在独立Docker容器中执行，防止恶意代码

4. 核心功能模块

4.1 用户模块

  • 注册/登录（JWT鉴权）
  • 角色管理（学生/教师/管理员）
  • 个人信息管理、提交记录查看

4.2 题目模块

  • 题目CRUD（题干、题解、标签、分类）
  • 难度分级（简单/中等/困难）
  • 题目搜索与筛选
  • 测试用例管理

4.3 判题模块

  • 异步判题流程：用户提交 → RabbitMQ → 沙箱拉取镜像 → 编译运行 → 对比输出 → 返回结果
  • 判题结果：AC/WA/TLE/MLE/CE/RE
  • 支持自定义时间/内存限制
  • 特殊判题（SPJ）

4.4 代码沙箱模块（独立服务）

  • Docker容器生命周期管理
  • 资源限制（CPU、内存、网络隔离）
  • 安全策略（禁止系统调用、文件访问限制）
  • 超时自动销毁

4.5 比赛模块

  • ACM/IOI赛制支持
  • 实时排行榜
  • 比赛数据统计

5. 关键技术实现

5.1 Docker沙箱判题引擎

核心安全措施：

  • 使用 --network=none 禁止网络访问
  • --memory 和 --cpus 限制资源使用
  • --read-only 只读文件系统
  • --security-opt=no-new-privileges 防止提权
  • 使用非root用户执行代码

5.2 MySQL主从读写分离

  • 主库：处理写操作（INSERT/UPDATE/DELETE）
  • 从库：处理读操作（SELECT）
  • 通过Spring AOP + 自定义注解 @ReadOnly 实现自动路由
  • 使用Binlog实现数据同步

5.3 Redis Sentinel高可用

  • Sentinel集群监控Redis主从状态
  • 主节点故障时自动故障转移
  • 用于：会话管理、题目缓存、排行榜实时计算、分布式锁（防止重复提交）

5.4 RabbitMQ异步判题

  • 死信队列处理超时任务
  • 消息持久化防止丢失
  • 手动ACK确认机制
  • 根据语言类型路由到不同队列

核心表结构

用户表 (user)

user_id BIGINT 用户ID

created_at DATETIME 提交时间

7. 部署方案

部署架构

CI/CD流水线

  1. 代码推送 → 触发Webhook
  2. 构建阶段：Maven编译打包
  3. 测试阶段：运行119个单元测试
  4. 镜像构建：Docker Build各服务镜像
  5. 部署阶段：Docker Compose一键部署/滚动更新

8. 性能优化

性能指标

  • 日均判题量：1000+次
  • 并发判题：支持多容器并行判题
  • 服务用户：500+学生

优化措施

优化项 方案 效果
读写分离 MySQL主从 + AOP路由 读性能提升50%+
缓存 Redis缓存热点题目/排行榜 DB查询减少60%+
异步判题 RabbitMQ削峰 高峰期不阻塞
静态资源 Nginx + Gzip 前端加载<1s
连接池 HikariCP 减少连接开销
