五、TensorRT（模型加速推理）

为什么需要 TensorRT

训练好的模型（PyTorch/ONNX）推理速度慢，TensorRT 通过图优化、精度校准（FP16/INT8）、内核自动调优大幅加速。

六、迁移学习实战

核心思路：用别人训练好的模型（预训练权重），只微调最后一层适配你的任务。

为什么迁移学习有效？ 预训练模型已经学会了提取边缘、纹理、形状等通用特征，你只需要让它学会识别你的特定目标。

七、模型评估指标

指标 公式 含义
IoU 交集面积 / 并集面积 预测框和真实框的重合度，>0.5 算检测到
Precision TP / (TP + FP) 预测为正的有多少是对的（不要误报）
Recall TP / (TP + FN) 真实为正的有多少被找到（不要漏报）
F1 2×P×R / (P+R) Precision 和 Recall 的调和平均
mAP 各类别 AP 的均值 目标检测的核心指标，越高越好

指标速记

  • Precision 高 → 宁可漏掉也不要误报（如：人脸支付）
  • Recall 高 → 宁可误报也不要漏掉（如：癌症筛查）
  • mAP > 0.5 → 基本可用
  • mAP > 0.7 → 效果不错
  • mAP > 0.9 → 优秀

八、最佳实践

  1. 数据为王：标注质量 > 数据量 > 模型选择
  2. 先跑通再优化：用 YOLOv8n 验证流程，再换大模型提精度
  3. 数据增强：旋转、翻转、亮度调整、Mosaic 拼接
  4. GPU 必需：训练必须有 GPU（推荐 RTX 3060+），推理 CPU 也行
  5. 部署优先 ONNX：跨平台兼容性最好

九、常见问题

Q: 少量数据能训练吗？
A: 可以，用迁移学习 + 数据增强，100-500 张标注图片即可微调。

Q: YOLOv5 还是 YOLOv8？
A: 新项目用 v8（ultralytics 维护，API 更简洁），老项目维护用 v5。

Q: 检测效果不好怎么办？
A: 检查标注质量 → 增加训练数据 → 调整 anchor size → 换更大的模型。
