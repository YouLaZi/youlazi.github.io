代码审查要点

  • 功能是否正确实现
  • 是否有潜在 bug / 边界情况
  • 代码风格是否一致
  • 是否有性能问题（N+1 查询、大循环）
  • 测试是否覆盖关键路径

六、Maven vs Gradle

维度 Maven Gradle
配置 XML（pom.xml） Groovy/Kotlin（build.gradle）
构建速度 慢 快（增量编译 + 构建缓存）
灵活性 约定优于配置 高度灵活
学习曲线 低 中
Spring Boot 支持 ✅ ✅
推荐场景 传统企业项目 新项目

八、测试

关键指标：TPS（每秒事务数）、响应时间（P99 < 500ms）、错误率（< 1%）

九、最佳实践

  1. Vue 项目：用 Composition API + Pinia + TypeScript
  2. React 项目：用函数组件 + Hooks + Zustand
  3. Docker：多阶段构建，用 Alpine 基础镜像减小体积
  4. Git：小步提交、频繁合并、写好 commit message
  5. 测试：单元测试覆盖核心逻辑，集成测试覆盖关键流程

十、常见问题

Q: Vue 和 React 学哪个？
A: 国内企业 Vue 多，海外 React 多。建议先精通一个，另一个了解即可。

Q: TypeScript 必须学吗？
A: 项目规模大了之后 TS 基本是标配，建议直接从 TS 开始。

Q: CI/CD 用 GitHub Actions 还是 Jenkins？
A: 个人/小团队 → GitHub Actions（免费、配置简单）；企业 → Jenkins（灵活、生态成熟）。
