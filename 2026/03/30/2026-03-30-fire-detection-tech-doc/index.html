工业级火灾智能监测系统 技术文档

1. 项目概述

项目背景

传统火灾监测依赖烟雾传感器，存在误报率高、响应慢、覆盖范围有限等问题。本项目利用深度学习目标检测技术，对视频流进行实时火灾识别，实现毫秒级响应的智能火灾监测。

项目目标

  • 训练高精度火灾检测模型（mAP@0.5 ≥ 98%）
  • 实现TensorRT加速推理（<500ms/帧）
  • 支持大规模视频并发监控（99+路）
  • 提供完整的告警链路（Web通知 + 邮件 + 短信）

项目成果

  • 🏆 国家级大创项目
  • 🏆 中国计算机设计大赛国家级三等奖
  • mAP@0.5 达 98%+
  • TensorRT推理 <500ms
  • 支持 99+路视频并发监控
  • 训练样本 12000+

4. 核心功能模块

4.1 视频流处理模块（camera.py）

  • RTSP流接入：支持海康、大华等主流摄像头
  • 多路并发：多线程/多进程处理99+路视频流
  • 抽帧策略：可配置抽帧频率，平衡精度与性能
  • 断线重连：自动检测流中断并重连

4.2 火灾检测模块（YoloV5）

  • 目标类别：火焰(fire)、烟雾(smoke)
  • 实时检测：逐帧推理，输出边界框和置信度
  • 区域过滤：可设置ROI区域，排除干扰
  • 灵敏度调节：置信度阈值可动态调整

4.3 告警模块

  • 邮件告警：SMTP发送告警邮件，附带截图
  • 短信告警：短信API推送告警信息
  • Web推送：WebSocket实时推送到监控大屏
  • 告警记录：历史告警查询与统计
  • 告警升级：持续告警自动升级通知级别

4.4 Web管理前端（fire\Web）

  • 监控大屏：多画面实时预览
  • 告警看板：实时告警列表与统计
  • 设备管理：摄像头增删改查
  • 系统配置：检测参数、告警规则配置

6. 数据库设计

核心表结构

摄像头表 (camera)

字段 类型 说明
id INT 主键
name VARCHAR(100) 摄像头名称
rtsp_url VARCHAR(500) RTSP流地址
location VARCHAR(200) 安装位置
status TINYINT 状态(0离线/1在线)

告警记录表 (alert)

字段 类型 说明
id BIGINT 主键
camera_id INT 摄像头ID
alert_type VARCHAR(20) 告警类型(火焰/烟雾)
confidence FLOAT 置信度
snapshot VARCHAR(500) 告警截图路径
alert_time DATETIME 告警时间
is_handled TINYINT 是否已处理

7. 部署方案

  • 推理服务器：NVIDIA GPU服务器（TensorRT需要CUDA）
  • Web服务：Flask + Gunicorn + Nginx
  • 摄像头：RTSP协议接入，局域网/互联网均可
  • 告警通道：邮件(SMTP) + 短信(API)

8. 性能优化

性能指标

指标 数值
mAP@0.5 98%+
单帧推理 <500ms (TensorRT FP16)
并发视频路数 99+

优化措施

  • TensorRT FP16量化：推理速度提升2x
  • 多线程视频流处理：充分利用多核CPU
  • 抽帧策略：非关键帧跳过检测
  • 模型轻量化：YoloV5s模型，参数量仅7.2M
